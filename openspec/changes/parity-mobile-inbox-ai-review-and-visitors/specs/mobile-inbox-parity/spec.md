## ADDED Requirements

### Requirement: Mobile inbox MUST expose AI workflow context for conversations

Mobile inbox list and conversation views SHALL display AI workflow context, including handoff state and relevant reason metadata when available.

#### Scenario: Conversation is in AI handoff state

- **WHEN** mobile inbox loads a conversation with AI handoff state
- **THEN** conversation UI SHALL display handoff status and reason context

### Requirement: Mobile conversation view MUST distinguish AI-generated and human-agent messages

Mobile message rendering SHALL clearly indicate whether non-visitor messages were generated by AI or by a human agent.

#### Scenario: Conversation contains both AI and human replies

- **WHEN** mobile conversation message list is rendered
- **THEN** AI messages SHALL be visually attributed as AI
- **AND** human-agent messages SHALL remain distinguishable

### Requirement: Mobile app MUST provide visitor directory list/search/detail navigation

Mobile app SHALL provide visitor directory navigation with list/search and detail views for operator workflows.

#### Scenario: Operator searches visitor directory

- **WHEN** operator enters visitor search text
- **THEN** mobile directory SHALL filter results by supported visitor identity fields
- **AND** selecting a result SHALL navigate to visitor detail view
