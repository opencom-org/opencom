## ADDED Requirements

### Requirement: Web and Convex MUST use one canonical numbered visitor-readable-ID generator

Numbered fallback visitor-readable IDs SHALL be generated by one shared implementation consumed by both web and Convex surfaces.

#### Scenario: Numbered fallback generation for the same visitor ID

- **WHEN** web and Convex generate a numbered readable ID for the same `visitorId`
- **THEN** both surfaces SHALL produce the same adjective-noun-number output

### Requirement: Numbered visitor-readable-ID generation MUST remain deterministic

The shared formatter SHALL return stable output for a given `visitorId` with fixed dictionaries and algorithm salts.

#### Scenario: Deterministic output is requested repeatedly

- **WHEN** the formatter is invoked multiple times with the same `visitorId`
- **THEN** it SHALL return the same output each time

### Requirement: Web visitor identity fallback precedence MUST remain unchanged

Web identity label formatting SHALL continue to prioritize name, then email, then persisted readable ID, then deterministic fallback from visitor ID.

#### Scenario: Visitor record has no name/email/readableId

- **WHEN** web formats a visitor identity label with only `visitorId`
- **THEN** it SHALL return the shared deterministic numbered readable ID
